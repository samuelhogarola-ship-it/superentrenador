# ROADMAP

## Para que sirve este documento

Sirve para decidir que entra en cada fase, evitar mezclar ideas con urgencias, y poder delegar trabajo sin tener que reconstruir la vision del producto cada vez.

## Fase 0 · Base ya creada

- repo independiente
- estructura Vite + React + TypeScript
- migraciones y seeds iniciales
- landing con acceso
- modo demo
- clientes, plantillas, rutina y PDF base

## Fase 1 · MVP funcional real

- Supabase Auth real
- trainer profile real
- CRUD real de clientes
- CRUD real de training plans
- biblioteca global legible desde DB
- logos reales en Storage
- deploy en `app.superentrenador.com`

## Fase 2 · Calidad de producto

- editor de rutinas mas claro
- ejercicios compuestos completos
- selector visual de ejercicios
- thumbnails o ilustraciones placeholder
- mejoras responsive del panel
- mejor experiencia de impresion

## Fase 3 · Operativa del entrenador

- plantillas privadas completas
- duplicar rutinas entre clientes
- versionado simple de planes
- notas internas del entrenador
- historial de cambios del cliente

## Fase 4 · Portal cliente

- acceso cliente
- visualizacion de rutina
- seguimiento basico
- registro de cargas / RM
- futuro soporte de video o imagenes por ejercicio

## Fase 5 · Capa comercial grande

- web madre de Superentrenador
- onboarding comercial
- pricing / planes
- captacion de entrenadores
- panel admin o soporte interno
